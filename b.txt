You are thinking about the **right thing** now.
Forget Python for a moment — what you described is essentially an **API contract design problem**.

Your requirement is basically:

> Send a document → AI extracts data → Return structured JSON

So the key things to understand are:

1. **Request format**
2. **Response format**
3. **Error handling**
4. **Processing behavior (sync/async)**
5. **Limits & constraints**

Let’s walk through this like an API architect.

---

# 1️⃣ Typical Request Structure

Most document AI APIs use **multipart file upload**.

Example endpoint:

```
POST /invoice/extract
```

### Request (multipart/form-data)

Example:

```http
POST /invoice/extract
Content-Type: multipart/form-data
```

Body:

```
file: invoice.pdf
documentType: invoice
language: en
```

### Example with CURL

```bash
curl -X POST https://xyz-ai/extract \
  -F "file=@invoice.pdf"
```

---

## Possible Request Fields

| Field         | Purpose              |
| ------------- | -------------------- |
| file          | PDF or image         |
| documentType  | invoice, receipt etc |
| vendorHint    | optional vendor hint |
| language      | OCR language         |
| correlationId | tracking             |

Example full request:

```json
multipart/form-data
file = invoice.pdf
documentType = invoice
correlationId = 12345
```

# 5️⃣ Possible Processing Models

This is a **very important design question**.

### Model A — Synchronous

Client sends document
Response returned immediately.

```
POST /extract
→ returns JSON
```

Good for:

* small documents
* fast models

---

### Model B — Asynchronous

Used if processing takes long.

Flow:

```
POST /extract
→ returns jobId
```

Then:

```
GET /extract/{jobId}
→ returns result
```

Example response:

```json
{
  "jobId": "abc123",
  "status": "processing"
}
```

---

# 6️⃣ File Constraints

You should ask about:

* Max file size
* Supported file formats
* Page limits
* Image resolution limits

Example:

| Type | Supported |
| ---- | --------- |
| PDF  | yes       |
| JPG  | yes       |
| PNG  | yes       |
| TIFF | maybe     |

---

# 7️⃣ Security / Compliance

In a bank environment you must ask:

* Is the document stored?
* Is it deleted after processing?
* Is encryption required?
* Is authentication required?

Example:

```
Authorization: Bearer token
```

---

# 8️⃣ Questions You Should Ask About the Endpoint

These will make you look very sharp in discussion.

### Request Questions

* What document formats are supported? (PDF, JPG, TIFF)
* What is the max file size?
* Is the API synchronous or asynchronous?
* Do we send file directly or via storage URL?

---

### Response Questions

* What fields are extracted?
* Are confidence scores included?
* Is the response schema fixed or evolving?
* How are line items represented?

---

### Error Handling Questions

* What happens if extraction fails?
* Are partial results returned?
* How are invalid documents handled?

---

### Performance Questions

* Typical processing time per invoice?
* Rate limits?
* Max concurrent requests?

---

### Integration Questions

* Is OpenAPI / Swagger available?
* Is there a sandbox environment?
* Are sample responses available?
